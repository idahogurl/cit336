Deployment

By now you are wondering how to get your files to our web host. This process is called deployment. There are several methods to send files to a remote server with File Transfer Protocol (FTP) and SSH (Secure Shell) being most common.

FTP (File Transfer Protocol)

SSH (Secure Shell)

The downside to using an FTP client or SSH client is that you manually have to upload your web site with your changes. It becomes tedious to upload every little change when you already have to push your changes to GitLab. That's when Continuous Deployment comes into play.

Continuous Deployment

What if every time you committed or merged that your web site was updated with the changes? Look no further than to Continuous Deployment (CD). GitLab provides built-in Continuous Deployment. Continuous deployment is an automated process to deploy your web site changes. Your repository has been setup for is already setup to deploy each time a commit is pushed to "master."

To implement Continuous Deployment, you need a script file. This script file contains an ordered set of commands to tell GitLab how to deploy your changes. You should already have this file (.gitlab-ci.yml) inside your repository.
