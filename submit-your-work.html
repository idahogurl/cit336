Introduction

We will be using Git to submit assignments. You will still be required to submit your assignment. Instead of a ZIP file of your code, you will provide your video URL within a document or text file. The submission date of that file will be used to determine whether the assignment was late. To use Git, we have GitLab hosting our repositories. GitLab, Bitbucket, GitHub are the most well known Git hosts. Git is a version control system. For those unfamiliar with Git, here is a video that gives a high-level explanation.




So, Git provides:

  • multiple individuals with the ability to work on the same code
  • a timeline of saves called commits, where each has a description and date

Branching

The way Git implements both functionalities is through the use of branches. When a repository is created, a branch called "master" is also created. From "master", individuals create new branches. Sometimes branches are discarded, but usually, the branch changes get merged into the master branch. Merge conflicts do arise, but that is a topic for another day.

You will create a branch for every assignment. Instead of having you merge into and branch from "master," you will branch from the previous assignment's branch. That is not typically the procedure developers follow. With 25 merge requests to review, I can't review them fast enough for you start the next assignment from "master". If you want to learn more about branching and merging, you can read this article.

Merge Requests

A Merge Request is merely a request to merge your branch of code changes into another branch; usually, this will be the master branch. GitHub uses the term Pull Request instead, but are essentially the same.

Merge requests within GitLab have many features available. We are only concerned about the ability to review and discuss the proposed changes. You will have your code changes reviewed by me, and then I will close the merge request. Usually, I'd merge your changes into "master" before closing it but since we are branching from the previous assignment there is no need.

Note: You do not have to wait for your merge request to be reviewed for you to start the next assignment. Just create a new branch after you create the merge request, and you are ready to go. At the end of the semester, you are welcome to merge your final project's branch into "master."
