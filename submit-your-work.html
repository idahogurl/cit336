To use Git, we have GitLab hosting our repositories. GitLab, Bitbucket, GitHub are the most well known Git hosts. Git is a version control system. For those unfamiliar with Git, here is a video that gives a high-level explanation.




So, Git provides:

  1. the ability for multiple individuals to work on the same code
  2. a timeline of saves called commits, where each has a description and date

Branching

The way Git implements the first functionality is through the use of branches. When a repository is created, a branch called "master" is automatically created. From "master", individuals create new branches. Sometimes branches are discarded, but usually, the branch gets merged into "master" by the merge request approver once approved.

Sometimes you cannot merge because of competing changes (your changes vs. another person's changes to the same file) that Git cannot resolve without help. Since you are the only developer working on your code this should rarely happen. If it does happen, please contact your instructor for help. Resolving merge conflicts is beyond the scope of this course.

Before you start an enhancement, you must create a branch as outlined in the video and steps below. As you work, you will commit your changes to that branch. will not be merging your branch into "master" as described above. Instead, you will just repeat the branch creation steps. That is not typically the procedure developers follow. With 25 merge requests to review, I can't finish them fast enough.

Merge Requests

A Merge Request is merely a request to merge your branch of code changes into another branch; usually, this will be the "master" branch. GitHub uses the term "Pull Request" instead, but their purpose is the same.

Merge requests give the ability for another individual to review and discuss the proposed changes. You will have your code changes reviewed by me, and then I will merge your branch into the "master" branch. The creation date of the merge request serves as your assignment submission date. If your merge request is created after the due date then it is late.

Do not wait for your merge request to be merged into "master" for you to start the next assignment. Just create a new branch after you create the merge request, and you are ready to go. At the end of the semester, you should have 11 branches since we won't be deleting branches after merge.
