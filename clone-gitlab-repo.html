Change Notifications

Steps

  1. Log into your GitLab account. If you are already logged go to the upper-right corner and click on your profile picture.
  2. Once the menu appears click "Settings".
  3. Locate the sidebar on the left and click "Notifications".
  4. In the right panel, click on the "Global" dropdown belonging to your "acme" repository.
  5. Select the "Custom" selection. Modify the selections until only "Close merge request" and "Merged merge request" are checked.

Clone a GitLab Repository

This document outlines the steps needed to create a local repository from your GitLab repository. That means you are going to create a copy of your GitLab repository locally so you can make changes. The term for this process is cloning. You will use Visual Studio Code to clone your "acme" repository.

Prerequisites:

  • XAMPP (Windows or Mac)
  • GitLab Account
  • Visual Studio Code

Add SSH Key

Use an existing SSH Key

Windows

  1. Open the Windows explorer and navigate to C:\Users\your_username\.ssh
  2. Right-click on id_rsa.pub and click on "Open". Skip to "Create a New Key" below if file does not exist.
  3. Choose Notepad when asked which program to open the file.
  4. Copy the entire file contents.

Mac

  1. Open a terminal window.
  2. Type cat ~/.ssh/id_rsa.pub
  3. Skip to "Create a New Key" below if nothing is output
  4. Copy the entire output.

Create a New Key

  1. Open a terminal window. In Windows, press Win+R to open the Run dialog. Type cmd and press Enter.
  2. Type ssh-keygen -o -t rsa -b 4096 -C "email@example.com"
    Replace email@example.com with your email address.
  3. Press the "Enter" key until the key is generated.

  4. Windows

    1. Open the Windows explorer and move to C:\Users\your_username\.ssh
    2. Right-click on id_rsa.pub and click on "Open".
    3. Choose Notepad when asked which program to open the file.
    4. Copy the entire file contents.

  5. Mac

    1. Type cat ~/.ssh/id_rsa.pub
    2. Skip to "Create a New Key" below if nothing is output.
    3. Copy the entire output of the file's contents.

Paste into GitLab

  1. Log into your GitLab account. If you are already logged go to the upper-right corner and click on your profile picture.
  2. Once the menu appears click "Settings".
  3. Locate the sidebar on the left and click "SSH Keys".
  4. Paste your SSH Key into the textbox.
  5. Enter a title.
  6. Click "Add key" button.

Copy GitLab repository URL

  1. Log into your GitLab account. If you are already logged go to the upper-left corner and click the GitLab logo. This will take you to your project list.
  2. When the project list loads click on your "acme" project.
  3. To the right, click the blue "Clone" button
  4. A small popup will display. Click the copy button next to the 1st textbox (Clone with SSH).
  5. You now have the URL you need to connect Visual Studio Code to GitLab.

Notice!

Those familiar with Git having used GitHub may try to push a new file to GitLab to test the connection. Do not attempt this. An error message will result since the master branch is protected. The master branch may only be modified by merged merge requests.

Test

  1. Ensure XAMPP is running.
  2. Open your web browser and enter http://localhost/[BYU-I username]-acme
  3. Once loaded you should see a listing of your repository contents.

Troubleshooting

When connecting VS Code to GitLab, you may get the error command git.clone not found. This means that Git software needs to be installed. The installers mentioned in the videos are available at http://www.git-scm.com. Note: After the installation finishes, you need to restart Visual Studio Code.
